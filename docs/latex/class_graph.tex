This functions uses the Ant Colony Optimization algorithm to find the best path in a given distance matrix. Calculates the Minimum Spanning Tree (MST) of a graph using Prim\textquotesingle{}s Algorithm. 

This function calculates the Minimum Spanning Tree (MST) of a given graph using Prim\textquotesingle{}s Algorithm. To begin, the function initializes necessary data structures such as \char`\"{}parent\char`\"{}, \char`\"{}dist\char`\"{}, \char`\"{}visitied\char`\"{} and a priority queue named \char`\"{}pq\char`\"{}.

The vector \char`\"{}parent\char`\"{} stores the parent node and the corresponding edge weight for each node in the MST. The vector \char`\"{}dist\char`\"{} stores the minimum distance from the MST to each node. The vector \char`\"{}visited\char`\"{} keeps track of the nodes that have been visited. The priority queue \char`\"{}pq\char`\"{} is used to select the next node with the minimum distance.

The functions starts constructing the MST from node 0 by setting its parent as -\/1 and the respective distance to 0. and adds this initial node to the priority queue. While the priority queue is not empty, the function extracts the node with the minimum distance from the queue. If it has been visited then it skips it, otherwise it marks the node as visited and iterates over its adjacent edges. For each unvisited adjacent node, it updates the minimum distance and parent if a shorter edge is found. After that, the updated nodes are added to the priority queue.

After the MST is constructed, the function creates a new adjacency list \char`\"{}mst\char`\"{} for the edges belonging to the MST. Then the function iterates over the \char`\"{}parent\char`\"{} vector and adds the edges to the adjacency list \char`\"{}mst\char`\"{}. Note that \char`\"{}mst\char`\"{} is a bidirectional graph, so both ends of an edge are added to their respective nodes.

After this, the functions returns the adjacency list \char`\"{}mst\char`\"{} representing the MST of the \mbox{\hyperlink{class_graph}{Graph}}.

It does this by initializing the temperature \char`\"{}t\char`\"{} to a high value and generating a random initial permutation of size \char`\"{}n\char`\"{} using the generate\+Random\+Permutation() function. After this, the function calculates the initial cost of the permutation using the \mbox{\hyperlink{class_graph_aedf8896f598856ccb7e59b580c9ae03b}{get\+Value()}} function. It then iteratively performs the simulated annealing process until the temperature \char`\"{}t\char`\"{} reaches zero. In each iteration, the function applies the \mbox{\hyperlink{class_graph_a3e0530df9057911b527116678c3fce82}{get\+Sample()}} function to generate a neighbouring solution and updates the cost \char`\"{}value\char`\"{}. After this, the function returns the best solution (represented by \char`\"{}ans\char`\"{}) that it found.

Solves the Traveling Salesman Problem using a Backtracking Algorithm. 

This functions uses a Backtracking algorithm to solve the Traveling Salesman Problem (TSP). It does this by initializing a vector \char`\"{}v\char`\"{} that is used to keep track the visited nodes where \char`\"{}v\mbox{[}i\mbox{]}\char`\"{} is true if the node represented by \char`\"{}i\char`\"{} has been visited. The functions uses recursion to explore all the possible paths from the \char`\"{}curr\+Pos\char`\"{} and updates the \char`\"{}best\+Path\char`\"{} and its respective cost \char`\"{}ans\char`\"{} if a shorter path is found. The algorithms uses pruning techniques to optimize the search, avoiding unnecessary exploration of paths with a higher cost than the current best path.
